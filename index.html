redis(2)-深入了解redis持久化机制

2020-05-12
#redis
本文系作者原创，如需转载请注明链接

　　什么是持久化（Persistence）？个人理解，持久化就是将数据（如内存中的数据）保存到永久性存储的设备（磁盘）中。对于redis数据库而言，由于在工作时，其数据都存在内存中，一旦发生断电、宕机等不可预知的灾难，redis中的数据就会全部丢失，因此，在redis中，持久化数据就显得十分必要了。

redis的持久化机制：

1.RDB（redis database）：每隔一段时间，把内存中的数据写入磁盘的临时文件，作为快照，恢复的时候把快照文件读进内存。如果宕机重启，那么内存里的数据肯定会没有的，那么再次启动redis后，则会恢复。

2.AOF（append only file）：AOF会以日志的形式来记录服务器接收到的写操作，记录的方式是以文件追加的方式记录，当服务器重启时，AOF执行的操作就是把追加的文件从开始到结尾读取执行写操作。

RBD的优点与缺点：

  • 优点
  1. 每隔一段时间备份，全量备份
  2. 灾备简单，可以远程传输
  3. RDB最大限度地提高了redis的性能，对于备份，父进程需要做的唯一一件事就是fork()子进程，备份的工作全部由子进程完成，主进程不会有任何io操作（不会有写入修改或删除），保证备份数据的的完整性
  4. 相对AOF来说，当有更大的数据集文件时可以快速重启恢复
  • 缺点
  1. 发生故障时，有可能会丢失最后一次的备份数据
  2. RDB通过fork()子进程的方式进行备份，当数据集很大时，fork()可能会很耗时，对CPU造成负担，甚至有可能导致Redis停止为客户端服务几毫秒甚至一秒钟

AOF的优点与缺点：

  • 优点
  1. AOF的持久化机制比RDB更好，可以设置不同的fsync策略，redis提供了三种fsync策略：永不fsync、每隔一秒fsync、每个命令fsync，实际生产中一般以秒为单位进行备份，如果发生问题，也只会丢失最后一秒的数据，大大增加了可靠性和数据完整性。
  2. 当AOF日志文件太大时，redis会在后台自动执行重写机制，重写是完全安全的，不会影响客户端的读写操作。
  3. AOF 日志包含的所有写操作，会更加便于redis的解析恢复。
  • 缺点
  1. 对于同一个数据集，AOF文件通常大于等效的RDB文件。
  2. 根据确定的fsync策略，AOF可能会比RDB慢，因此，通常将fsync设置成秒级。

AOF持久化设置：

　　总结：如果能够接受一段时间的缓存丢失，那么可以使用RDB；如果对实时数据比较关心，则可以使用AOP，当然，如果从辩证的角度看待问题，肯定是一起结合使用最好，用RDB做冷备份（每隔一段时间备份一次），用AOF做热备份（每隔1s做一次备份）。 　　

　　redis在4.0版本后实现了混合持久化机制，将RDB备份文件与增量的AOF日志文件存在一起，这里的AOF日志不再是全量的日志，通过这样，兼容解决了RDB的缺点和AOF的缺点，在redis重启的时候，可以先加载rdb文件到内存中去，然后再将增量的AOF命令追加写入。

0基础快速搭建个人博客

2020-05-05
#搭建博客

　　最近想搭建一个自己的个人技术博客，期间踩了不少坑，因此想写一篇博文，帮助大家更好地建站。目前比较主流的静态网页生成工具有wordpress、jekyll、hexo、hugo、pelican、vuepress等。下面将介绍一下具体的区别。

　　wordpress是基于php语言编写的，使用wordpress搭建的博客是动态博客。与动态博客相对应的肯定是静态博客，jekyll、hexo、hugo等工具生成的网站博客都是静态的。个人理解动态与静态的区别就在于内容是否可以修改、管理。

　　除了wordpress，其余的都是静态博客生成工具，不同的是这些框架的开发语言不同。jekyll是基于ruby语言编写的，hexo是基于nodejs编写的，hugo是基于go语言编写的，pelican是基于python编写的，vuepress是vue实现的，其余的框架也都大同小异。

下面我们将基于hexo快速搭建一个属于自己的博客。

1. 安装nodejs
　　由于hexo是基于nodejs实现的，因此需要安装nodejs，如果电脑上已经有nodejs环境，这一步可以忽略。首先去nodejs官网下载对应的安装包，推荐下载v10.x版本，如果nodejs版本过高会有不兼容问题。https://npm.taobao.org/mirrors/node/v10.2.0/，以windows电脑为例，下载64位.msi安装包：
　　

然后安装nodejs，将路径选在D盘根目录下，当前，其他路径也是可以的，然后一路next，直到安装完成。

安装完后需要设置一下nodejs的相关配置，在刚才安装的目录下新建两个文件夹，node_global和node_cache，如下图所示：

node_global用于存放全局安装的模块，node_cache指定node缓存文件夹，然后打开cmd，执行指令：

npm config set prefix "D:\nodejs\node_global"

npm config set cache "D:\nodejs\node_cache"

若npm指令执行找不到，请检查nodejs环境变量是否正确。

2. 安装git

　　在git管网上安装git，如果电脑上已有git，这一步可以忽略，网址：https://git-scm.com/download ，以windows电脑为例，下载64位.exe可执行文件，然后运行。

一直点next直到安装结束，在命令行输入git --version检查git安装是否成功，如果git指令找不到，去系统环境变量path中增加git的安装目录。

3. 安装cnpm、hexo和hexo-deployer-git

　　由于npm安装插件是从国外的站点进行下载，速度很慢，因此可以安装cnpm替代npm，全局安装cnpm，并且指定淘宝镜像源：

npm install -g cnpm --registry=https://registry.npm.taobao.org

安装完成后检查cnpm指令是否正确，若cnpm指令找不到，在系统环境变量path中增加node_global目录，如下所示：

cnpm安装完成并且能够正确使用后，使用cnpm安装hexo：

cnpm install -g hexo-cli

安装完成后可使用hexo -v查看是否安装成功，如果成功了，再安装hexo-deployer-git：

cnpm install hexo-deployer-git
